# 长期备忘文件管理

## 触发条件

当用户有以下行为时加载本 skill：
- 要求保存某些内容但明确不希望每次对话自动加载
- 说"记一下"、"存起来"、"以后不会忘"
- 主动提议用文件代替 memory 存放内容
- 需要回顾之前保存的长期内容（说"读一下长期备忘"、提到之前存过的内容）

## 核心原则

**memory 只存指引，完整内容存文件。** 每次对话都会自动加载 memory，所以 memory 中只能放最精简的路径指针。详细内容放在外部文件中，只有需要时才读取。

- ✅ memory 中的指引 ≤ 50 字，格式固定："长期备忘存放在 <path>，按时间线追加内容。需要时由用户或agent读取，不自动加载。"
- ✅ 文件按时间线追加，每段标注日期和主题
- ✅ 需要回忆时读取文件
- ❌ 不要把 session 级别的具体内容塞进 memory
- ❌ 不要自动加载整个文件——只在用户要求或上下文明显需要时才读

## 文件结构规范

建议存放在 `~/longterm.md`，格式：

```markdown
# 长期备忘

此文件存放需要长期记住但不常加载的内容。每次按时间线追加，标注日期和主题。
需要查阅时告诉 agent 读取此文件即可。

---

## YYYY-MM-DD 主题标题

内容正文...

---

## YYYY-MM-DD 另一个主题

内容正文...
```

## 使用流程

### 保存新内容

1. 读取现有文件（如果文件已存在），确认有没有内容重复
2. 在文件末尾追加新章节：`---` 分隔 + `## 日期 主题` + 正文
3. 检查 memory 中是否已有指引条目；如果没有，添加一条轻量指引

### 检索内容

1. memory 中已有指引 → 按路径读取文件
2. 用户说"读一下长期备忘"或"你还记不记得我写过..." → 读取文件
3. 从文件中找到相关内容并展示

### 清理（可选）

- 如果文件中的某些内容已经过时（如已完成的项目计划、过期信息），在用户确认后可以删除相关章节
- 更优方案：保持追加，只在文件过大时做整理

## 注意事项与坑

- **最大的坑**：把 session 级别的具体内容（某次对话的详细产出、一次排查的日志、一次测试的结果）存进 memory。这些内容用 `session_search` 就能找回，不需要占 memory。
- **memory 空间敏感**：用户的 memory 上限约 2200 字符。指引条目必须保持在 50 字以内。
- **文件路径要统一**：一旦确定了路径（如 `~/longterm.md`），不要更换。否则 memory 中的指引就失效了。
- **追加而不是覆盖**：每次新增内容追加到文件末尾，保留历史记录。用户要求删除时再处理。
- **时效性内容不入文件**：几天后就会过期的信息（如临时笔记、一次性结论）不值得存文件。只保留值得跨月/跨年保存的内容。
