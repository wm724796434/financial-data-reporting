# 监管集市码表查询方法

> 实体文件编写 WHERE 条件时，每个条件的码值含义必须从监管集市表定义文件中查证，不能凭印象写。

## 查询步骤

### Step 1：定位监管集市表定义文件

项目中的监管集市表定义在 `参考资料/监管集市/` 目录下，命名规则：

```
<编号>_SMTMODS_<表名>_<中文名>.md
```

例如 `L_ACCT_LOAN` → `065_SMTMODS_L_ACCT_LOAN_贷款借据信息表.md`

查找方法：
```
ls 参考资料/监管集市/ | grep -i <表名关键词>
```

### Step 2：找到字段对应的码表编码

在每个表定义文件的"字段总表"中，`码表` 列标识了该字段使用的码表编码。
例如 L_ACCT_LOAN 的第8行：

```
| 8 | `ACCT_STS` | 账户状态 | ... | `A0005` 贷款账户状态 | ... |
```

→ ACCT_STS 使用码表 `A0005`（贷款账户状态）

### Step 3：在码表摘要中查码值

表定义文件末尾的"码表摘要"区域列出了所有引用的码表。搜索：

```
grep -A 15 '### A0005' 065_SMTMODS_L_ACCT_LOAN_贷款借据信息表.md
```

输出示例：
```
| 码值 | 名称 | 上级码值 | 描述 |
| `1` | 正常 | `A0005` | - |
| `2` | 逾期 | `A0005` | - |
| `3` | 结清 | `A0005` | - |
| `9` | 其它 | `A0005` | - |
```

### Step 4：写入 WHERE 条件的注释格式

```
条件  -- 码表<编码>：<码值>=<含义>（<为什么排除/保留>）
```

示例：
```sql
AND A.ACCT_STS <> '3'  -- 码表A0005：3=结清（排除），保留1=正常/2=逾期/9=其他
AND A.FUND_USE_LOC_CD = 'I'  -- 码表A0006：I=境内（中国大陆），排除O=境外（含港澳台）
AND C.CUST_TYP <> '3'  -- 码表C0003：3=个体工商户（排除），保留1=企业/2=政府机关…
AND A.CANCEL_FLG = 'N'  -- 码表A0010：N=否（未核销），排除Y=是（已核销）
```

## 本项目常用码表速查

| 码表编码 | 名称 | 常见字段 | 码值示例 |
|---------|------|---------|---------|
| A0004 | 贷款账户类型 | ACCT_TYP | 01=个人贷款、0102=个人经营性贷款、0301=贴现、90=委托贷款 |
| A0005 | 贷款账户状态 | ACCT_STS | 1=正常、2=逾期、3=结清、9=其他 |
| A0006 | 贷款资金使用位置 | FUND_USE_LOC_CD | I=境内（中国大陆）、O=境外（含港澳台） |
| A0010 | 是否标志 | CANCEL_FLG 等32个字段 | Y=是、N=否 |
| C0003 | 对公客户分类 | CUST_TYP | 1=企业、2=政府机关、3=个体工商户、4=社会团体… |

> 码值定义以监管集市表定义文件中的最新版本为准，此速查表仅作参考提示。
