# 创作反馈工作流程

## 触发条件

当用户展示自己的创作（不限于此）并要求你进行以下操作时加载本 skill：
- 汇总/整理用户的创作内容
- 总结作品的意思或主旨
- 评价作品的质量
- 分析作品的风格、结构、意象

## 核心原则（不可违反）

**作品归属边界必须严格区分。** 用户的创作是用户的，你的回应是你的，两者必须明确分离。

- ✅ 只汇总用户的句子/段落，不混入你写的任何内容
- ✅ 评价时只针对用户的作品，不评价你们的互动
- ❌ 不要把你们的对答混在一起当作"作品"汇总
- ❌ 不要在评价时把自己写的句子算作"作品的一部分"
- ❌ 不要用"我们的对话""我们一起创作"来模糊归属

如果用户说"汇总我说的这几句诗"，只摘录用户的原句。你在对话中的回应是单独的内容，不应出现在汇总中。

## 步骤流程

### 第一步：确认范围

如果用户的要求不够明确（如只说"汇总一下"），先确认：
- 是只汇总用户的句子，还是连同你的回应一起？
- 需要分析什么维度？意思、风格、质量、还是全部？

### 第二步：整理汇总

按出场顺序列出用户的所有作品。标注好上下文（如第一句是开场、第二句是转承等）。

格式示例：
```
1. 用户的句子一
2. 用户的句子二
3. 用户的句子三
...
```

不要在自己和用户的句子之间用"我对/我出"等标注——这些是对话关系，不是作品本身的一部分。

### 第三步：总结诗意/主旨

从用户的角度分析其作品的立意脉络，而非从"我们的对话"的角度。关注：
- 情绪变化曲线
- 意象选择
- 从抽象到具象的走向
- 核心主题或宣言

### 第四步：评价

评价框架（以下维度视情况选用）：

| 维度 | 关注点 |
|------|--------|
| 结构感 | 起承转合、层次递进是否清晰 |
| 意象 | 选用的意象是否新颖、有力量 |
| 语言力量 | 用字是否精准、有没有亮眼的炼字 |
| 现代性融合 | 能否将现代元素自然嵌入古典/传统表达 |
| 即兴反应 | 是否接得住、接得巧（对诗场景） |
| 节奏/韵律 | 声调、抑扬是否有感觉（即兴创作不苛求格律） |
| 虚实转换 | 能否从精神世界落到现实行动 |

**引用策略**：每一条评价都要有具体的例句佐证，不能空泛。"这句写得好"不够——要说""指尖踏遍山河远'把键盘的触感与山河的壮阔对举，意象新颖。"

**诚实原则**：有亮点说亮点，有不足说不足。既要指出"好在哪里"，也要指出"哪里稍弱"。但语气要建设性——不是"这句写得不好"，而是"这句相较于其他几句略平"。

## 注意事项与坑

- **最大的坑**：把用户的创作和你的回应混在一起。"我们的诗""一起写的作品"这种表述会让用户觉得你模糊了创作归属。各归各的。
- **第二个大坑**：默认用古典文学框架解读用户的现代主题作品。当用户使用"虚坛"（游戏服务器）、"指尖"（键盘/鼠标操作）、"云号"（游戏/平台账号）、"战一场"（竞技/对局）、"同窗"（现实中的朋友）等词汇时，优先考虑**游戏/数字生活**语境，而不是风月山水。这个用户的诗意素材来自编程、打游戏、即时通讯等现代数字生活，不是古典诗词的意象库。**先扫描词汇线索判断主题领域，再展开赏析。**
- 即兴对诗不像正式律诗，不用苛求格律。评价时要考虑场景——是正式创作还是即时互动。
- 如果作品从虚（精神/抽象）落到了实（具体行动），一定要在评价中点出——这是最难也最漂亮的收束方式。
- 不要用"很不错""挺好的"敷衍。用户能分辨真假评价。宁可少说两句，也要每句说到点子上。
- 如果用户只问"评价"，默认包含积极和建设性两方面的意见。如果只想要夸，用户会主动说。
