# 路由索引生成模式

> 批量源码分析后，生成双向交叉引用索引作为查询入口。
> 适用于有多层文档结构（源码→实体→参考）的项目。

## 三层查询架构

```
用户提问
  │
  ▼
第一层：路由索引（index.md）
  │  从表名查到程序列表
  │  从程序名查到实体文件链接
  ▼
第二层：实体文件（功能/概述）
  │  获取用途说明、功能概述、依赖关系
  ▼
第三层：源码文件（字段级）
    用户追问字段取数逻辑时 → 查SQL代码中的具体字段映射
```

## 索引结构

### 第一部分：按报表索引

```markdown
## <业务域>

### <中文名>（<接口表名>）

| 属性 | 值 |
|------|-----|
| 接口表 | `PBOCD_JS_201_XXX` |
| 实体文件 | [存量单位贷款_CLDWDK](实体/存量单位贷款_CLDWDK.md) |
| 涉及源码 | [bsp_sp_js_201_cldwdk.prc](../源码解析/加工层存储/...) 等N个 |
```

### 第二部分：按源码文件反向索引

```markdown
### <目录>

- [bsp_sp_js_201_cldwdk.prc](../源码解析/加工层存储/xxx.prc) → 
  [存量单位贷款_CLDWDK](实体/存量单位贷款_CLDWDK.md)、[单位贷款发生额_CLDWDK](实体/单位贷款发生额_CLDWDK.md)
```

## 生成脚本模式

```python
# 核心数据流
records = []  # (目录名, 文件标识, 显示名, 业务域, [表名列表], 实体相对路径, 源码相对路径, 用途)

# 第一步：遍历所有目录，提取程序→表映射
for d in DIRS:
    for fname in files:
        content = read_file(...)
        tables = extract_smtmods_tables(content)  # 需统一大写
        records.append(...)

# 第二步：构建反向索引
reverse_idx = {}  # {表名: [(显示名, 目录名, 实体路径), ...]}
for r in records:
    for t in r.tables:
        reverse_idx.setdefault(t, []).append((r.display_name, r.dir_name, r.ent_path))

# 第三步：生成 index.md
# 按目录分组输出第一部分
# 按表名排序输出第二部分

# 第四步：验证所有链接
import os, re
content = open("index.md").read()
for text, path in re.findall(r'\[([^\]]+)\]\(([^)]+)\)', content):
    if path.startswith("实体/") or path.startswith("源码解析/"):
        assert os.path.exists(path), f"Broken link: {path}"
```

## 踩坑记录

### 表名大小写分裂
- 源码中对 SMTMODS 表的引用可能大小写混用：`smtmods.l_cust_all` vs `SMTMODS.L_CUST_ALL`
- **必须**在提取时做 `re.IGNORECASE` + `.upper()`，否则反向索引中同表会分裂为不同条目

### 文件名括号处理（高频踩坑）

markdown 链接语法 `[text](path)` 中，半角括号 `()` 会被解析器截断。两种修复方案：

**方案一（推荐）：文件系统层面用全角括号**
实体文件名中的 `()` 替换为 `（）`：
```python
def safe_filename(s):
    return s.replace("(", "（").replace(")", "）").replace("/", "／")
```

**方案二：链接路径中 URL 编码**
仅用于链接路径，不影响实际文件名：
```python
def encode_md_path(p):
    return p.replace("(", "%28").replace(")", "%29")
```

两种方案选其一即可，**混用会导致路径不一致**。

### 实体文件路径检查
- 不要假设文件名 = 去掉扩展名 + `.md`。.sql 文件的文件名可能含特殊字符（括号、空格等）
- 验证时用 `os.path.exists()` 而非 glob 搜索

## 参考项目

- 金数源码项目：100个报表索引，52个实体文件（按接口表聚合），67个源码文件，111/123链接有效
- `首页/index.md` 作为一级路由，`AGENTS.md` 定义三层查询流程
- `实体/` 目录按接口表名组织，文件名格式：`<中文名>_<接口表名>.md`
- 中文名中的 `()` 必须使用全角 `（）` 避免链接截断
