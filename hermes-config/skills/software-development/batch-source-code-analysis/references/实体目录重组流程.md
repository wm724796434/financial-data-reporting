# 实体目录重组流程 — 从源码一对一向按报表聚合

## 场景

项目初期实体文件按源码文件一对一组织。当多个源码文件（加工层存储主程序 + 加工层特殊处理修正SQL + 应用层特殊处理汇总SQL）操作同一个接口表时，需按报表聚合重组。

## 数据收集脚本

`collect_reports.py` — 扫描源码文件，建立接口表→源码文件映射。

核心逻辑：

```python
# 匹配三种输出表语法
for m in re.finditer(
    r'(?:TRUNCATE\s+TABLE|INSERT\s+INTO|MERGE\s+INTO)\s+(?:\w+\.)?(PBOCD_[\w_]+|JS_[\w_]+|L_[\w]+)',
    content, re.I):
    output_tables.add(m.group(1).upper())
```

## 实体生成脚本

`generate_entities.py` — 按接口表生成聚合实体文件。

角色分类逻辑：
- 加工层存储 → 数据生成 (Producer)
- 加工层特殊处理 → 数据修正 (Corrector)
- 应用层特殊处理 → 数据引用 (Consumer)

旧实体内容合并：
- 用途说明 → 取主程序旧实体
- 关键逻辑 → 取主程序旧实体
- 修改历史 → 合并所有子文件，去重
- SMTMODS表引用 → 合并所有子文件，去重

## Index 生成脚本

`regenerate_index.py` — 重写 index.md，链接指向新的实体路径。

实体文件路径格式：`实体/<中文名>_<接口表名>.md`
中文名中的 `()` 必须替换为全角 `（）`。
接口表名中的 `JS_` 前缀必须保留。

## 验证脚本

```python
# 遍历所有 [text](path) 链接，检查 os.path.exists()
links = re.findall(r'\[([^\]]+)\]\(([^)]+)\)', content)
for text, path in links:
    if path.startswith("实体/") or path.startswith("../源码解析/"):
        full_path = os.path.normpath(os.path.join(BASE, "首页", path if path.startswith("../") else ""))
        if not os.path.exists(full_path):
            print(f"失效: {path}")
```

## 踩坑记录

1. **MERGE INTO** 语法必须匹配，只匹配 TRUNCATE TABLE / INSERT INTO 会遗漏修正类SQL
2. **中文括号** `()` 在 markdown 链接中被解析器截断 — 必须用全角 `（）`
3. **JS_ 前缀** 在 short_name 和实体文件名中必须一致
4. **带日期的版本后缀**（`_20251201`）和备份后缀（`_SPOP_BK`）需要过滤
5. **实体文件名含 `/`**（如"存量再贴现/转贴现"）会变成目录分隔符 — 替换为全角 `／`
