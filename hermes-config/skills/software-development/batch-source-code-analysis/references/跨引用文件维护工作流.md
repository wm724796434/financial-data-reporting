# 跨引用文件维护工作流

在 金数源码 项目中，实体文件、原文文件、首页索引之间存在大量交叉引用（markdown 链接）。对文件做任何重命名/路径变更时，必须同步更新所有引用，否则链接断裂。

## 触发场景

- 用户要求"去掉文件名中的日期前缀"
- 用户要求"统一文件命名规范"
- 用户要求"将实体文件按新规则重新命名"
- 用户要求"移动文件到不同目录"

## 工作流：批量重命名 + 跨引用更新

### 第一步：确认变更范围

```bash
# 列出目标目录所有文件
ls -1 "参考资料/金融基础数据系统/"
# 确认变更规则（如 "去掉前导 YYYY-MM-DD-"）
```

### 第二步：批量重命名

对简单的前缀去除：

```bash
cd "参考资料/金融基础数据系统"
for f in YYYY-*.md; do
  new="${f#YYYY-MM-DD-}"
  mv "$f" "$new"
done
```

对复杂规则（需要映射表时），用 Python 脚本生成 rename map：

```bash
import re, os
for old in files:
    new = re.sub(r'^\d{4}-\d{2}-\d{2}-', '', old)
    os.rename(old, new)
```

### 第三步：更新所有 markdown 引用

**核心操作**——在项目所有 .md 文件中替换旧路径引用：

```bash
# 全局替换：在 index.md、实体/ 等目录中替换旧名→新名
# 简单模式：批量替换统一的前缀
find . -name "*.md" -not -path "*/金数源码/*" \
  -exec sed -i 's/YYYY-MM-DD-//g' {} +
```

**重要**：替换必须在**全项目**范围内执行，包括：
- `实体/*.md` — 每个实体文件的 `> **来源**：[xxx-原文.md]` 引用 + `原文：[xxx-原文.md]` 尾部链接
- `首页/index.md` — 如果首页也引用了原文文件（通常不直接引用）
- `参考资料/金融基础数据系统/README.md` — 如果存在

⚠️ 禁止仅重命名文件而不更新引用——会导致实体文件中的原文链接断裂。

### 第四步：验证

```bash
# 检查是否还有旧名残留
grep -r 'YYYY-MM-DD-' --include="*.md" . --exclude-dir=金数源码
# 检查新名文件是否存在
ls -1 参考资料/金融基础数据系统/ | grep -cP '^\d{4}-\d{2}-\d{2}-'
# 应返回 0
```

### 第五步：更新技能库中的命名约定（如有变更）

如果命名规范发生了永久性变化（如"从此去掉日期前缀"），必须同步更新技能 `batch-source-code-analysis` 中的：
- `SKILL.md` — `原文文件命名规范` 章节的示例和 `parse_original_filename` 函数
- `references/金融基础数据系统原文实体重建.md` — 如果里面有文件名示例

## 已知的应用场景

| 场景 | 操作 | 可能涉及的目录 |
|------|------|---------------|
| 去掉日期前缀 | `sed -i 's/2026-05-04-//g'` 全局替换 | 原文文件 + 实体文件 |
| 统一命名风格（如括号替换） | `sed -i 's/(/（/g; s/)/）/g'` 全局替换 | 任意 |
| 文件迁移到子目录 | 需逐个更新链接路径 | 实体/ → 实体/分类/ |
| 接口表名变更 | 需更新所有实体文件和首页 | 全项目 |
