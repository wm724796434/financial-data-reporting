## PDF补充：字段级表格提取（完整版）

当用户要求"通读PDF全文，所有内容一比一整理"时，需要用完整版提取——包括字段定义表格（序号、字段中文名称、长度、字段释义、填写规则）。

### 第一步：定位章节页码

```bash
for page in $(seq 1 184); do
  text=$(pdftotext -f $page -l $page -raw V2.1.pdf -)
  echo "$text" | grep -E "^[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+" | head -1
done
```

得到每个章节的起始页码（如 `3.2.3.1 担保合同信息` 在第55页）。

### 第三步：解析字段表

PDF表格在raw模式下的特点：**表头行被拆分**——`序号` 和 `字段中文名称 长度 字段释义 填写规则` 不在同一行。所以不能按表头查找，要按**章节标题**定位。

```python
def extract_fields(text):
    """从raw文本提取字段——基于章节定位，非表头定位"""
    lines = text.split('\n')
    
    # 找章节标题（如 "3.2.3.1 担保合同信息"）
    section_start = -1
    for i, line in enumerate(lines):
        if re.match(r'\s*\d+\.\d+\.\d+\.\d+\s', line):
            section_start = i
            break
    if section_start < 0: return []
    
    fields = []
    i = section_start + 1
    while i < len(lines):
        line = lines[i].strip()
        
        # 下一个章节标题结束
        if re.match(r'\d+\.\d+\.\d+\.\d+', line):
            break
        
        # 匹配字段行：序号 + 中文名 + 长度
        # 核心正则（已验证对金融规范PDF有效）：
        m = re.match(r'(\d+)\s+([\u4e00-\u9fff][\u4e00-\u9fff\w]*)\s+([\w.]+)', line)
        if m:
            seq, name, length = m.group(1), m.group(2), m.group(3)
            
            # 收集后续行作为字段释义和填写规则
            desc_lines = []
            i += 1
            while i < len(lines):
                nl = lines[i].strip()
                if re.match(r'\d+\s+[\u4e00-\u9fff][\u4e00-\u9fff\w]*\s+[\w.]+', nl):
                    break  # 新字段开始
                if re.match(r'\d+\.\d+\.\d+\.\d+', nl):
                    break  # 下一章节
                if nl and not nl.isdigit() and '金融基础数据采集规范' not in nl:
                    desc_lines.append(nl)
                i += 1
            
            fields.append({"seq": seq, "name": name, "length": length,
                          "desc": ' '.join(desc_lines).strip()})
            continue
        i += 1
    
    return fields
```

**关键正则说明**（基于实际PDF验证）：
- 字段行：`r'(\d+)\s+([\u4e00-\u9fff][\u4e00-\u9fff\w]*)\s+([\w.]+)'` — 序号必须是数字开头，字段名以中文字符开头，长度格式如 `C..18`、`C2`、`D20.2`
- 长度格式常见值：`C..18`（字符18位）、`C1`（1字符）、`C3`（3字符）、`D20.2`（数字20位含2位小数）
- 续行收集：字段的释义通常跨3-5行，需收集到下一个字段行或章节标题为止

```python
def get_cn(short):
    candidates = [short]
    if short.startswith("JS_"): candidates.append(short[3:])
    for s in list(candidates):
        pure = re.sub(r'^\d+_', '', s)
        if pure != s: candidates.append(pure)
    for c in candidates:
        if c in CN_TABLE_MAP: return CN_TABLE_MAP[c]
    return short
```

## 三层查询路由

```
第一层：index.md（一级路由）
  → 按业务域分组查报表 / 按明细条目查实体路径

第二层：实体文件（原文总结 + 源码关键逻辑 + 特殊处理规则）
  → 报表说明（精简总结1-2句）← 非原文复制
  → 关键逻辑（从源码SQL提取的CASE WHEN映射、功能描述）
  → 特殊处理规则（从加工层特殊处理SQL提取的业务规则）
  → 涉及源码文件（三段式角色分类：生成/修正/引用）
  → 引用的监管集市表 + 字段清单（指向原文，不复制字段表）

第三层：源码解析文件（字段级取数逻辑）
  → 查SQL代码中的字段映射
  → 结合参考资料/监管集市/下的表定义文档
```

用户明确纠正：*"实体文件不能跟原文一模一样，只是对原文的总结，以及源码内容的整理，外加源码的特殊处理逻辑"*

## 常见踩坑

1. **文件名括号截断**：markdown `[text](path)` 中 `()` 会被解析器提前截断 → 替换 `(` → `（`
2. **中文名含 `/`**：如"存量再贴现/转贴现" → 替换 `/` → `／` 或去掉
3. **JS_ 前缀一致性**：实体文件名和 index.md 链接中的短名必须用同一套逻辑，否则链接断裂
4. **原文字段清单不复制**：原文字段清单动辄数百行，实体文件中只指引路径不复制全文
5. **PDF两栏文本交错**：raw模式提取后需清理混入的右栏文本（如末尾的"存款同业存放"等杂音）
6. **正则限定符**：`r'(JS_[A-Z0-9_]+)'` 而非 `\w+`，避免匹配到中文
