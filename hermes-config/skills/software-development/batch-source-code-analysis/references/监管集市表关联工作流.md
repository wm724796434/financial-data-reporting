# 监管集市表关联工作流

> 批量源码分析时，将源代码中引用的 `SMTMODS.xxx` 表与监管集市表定义文档进行交叉引用。
> 适用于金数（PBOC金融基础数据报送）类项目。

## 目录结构约定

```
参考资料/监管集市/
├── 监管集市表清单.md          ← 索引文件：列出所有表的中文名、物理表名、字段数、文档链接
├── 001_SMTMODS_L_CUST_ALL_全量客户信息表.md   ← 表定义文件
├── 002_SMTMODS_L_CUST_IDENTIFY_客户证件信息表.md
└── ...
```

## 查询步骤（固定顺序，不可倒置）

### 步骤一：读表清单
`监管集市表清单.md` 包含完整的表映射表，格式如下：

| 序号 | 系统 | 物理表名 | 中文名 | 字段数 | 引用码表数 | 文档 |
| --- | --- | --- | --- | --- | --- | --- |
| 65 | 监管集市 | L_ACCT_LOAN | 贷款借据信息表 | 189 | 41 | [打开](065_SMTMODS_L_ACCT_LOAN_贷款借据信息表.md) |

从中可以确认：
- 表是否存在
- 文件全名（按"排序号_SMTMODS_物理表名_中文名.md"的固定命名规则）

### 步骤二：读表定义文件
表定义文件的固定结构：

```
# 贷款借据信息表
> 物理表名：`L_ACCT_LOAN`
> 所属系统：监管集市

## 1. 报表首页
| 项目 | 内容 |
| 物理表名 | `L_ACCT_LOAN` |
| 中文名 | 贷款借据信息表 |
| 主题 | 账户（ACCT） |
| 频率 | 日 |
| 字段总数 | 189 |

## 2. 字段总表
| 序号 | 字段名 | 中文名 | 类型 | 数据类型 | 长度 | 主键 | 可空 | 码表 | 取值说明 |
```

提取的关键信息：
- **中文名**（用于实体文档和源码头部注释中的"依赖关系"表格）
- **主题**（ACCT=账户, CUST=客户, AGRE=协议, TRAN=交易, FINA=财务）
- **频率**（日/月）
- **字段数**（衡量表复杂度）
- **关键字段**（源码 SQL 中引用的字段的中文名）

### 步骤三：匹配源码中的表引用
从源码中提取表引用的命令：
```bash
iconv -f GBK -t UTF-8 file.prc | grep -oP 'SMTMODS\.\w+' | sort -u
```

## 依赖关系表格生成

在实体文档和源码注释中使用统一格式：

```markdown
| 表名 | 中文名 | 在存储过程中的用途 |
|------|--------|-------------------|
| SMTMODS.L_ACCT_LOAN | 贷款借据信息表 | 主数据源：提取当月发放的贷款记录（DRAWDOWN_DT） |
| SMTMODS.L_CUST_C | 对公客户补充信息表 | 排除个体工商户（CUST_TYP <> '3'） |
```

要点：
- 每个表都必须有中文名（从表定义确认）
- "用途"列要写具体：说明在该存储过程中如何使用该表的哪个字段、做什么过滤
- 标注主数据源的表（FROM 子句的第一张表或主驱动表）
