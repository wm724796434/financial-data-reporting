# 字段映射行内注释技能

## 适用场景

为 `源码解析/加工层存储/` 目录下的 `.prc` 主程序文件添加字段映射注释，帮助快速定位"某个字段数据从哪里来、去了哪里"。

## 不适用场景

- `_his.prc` 历史版本文件（跳过）
- `应用层特殊处理/` 下的修正SQL（不直接生成接口表数据）
- `加工层特殊处理/` 下的校验SQL
- `outfile.sh` 等辅助文件

## 处理流程

### 第一步：确认目标文件

从 `源码解析/加工层存储/` 中选择非 `_his` 的主程序 `.prc`。

### 第二步：读取文件头部注释

读取文件的前30-50行，获取：
- 程序用途注释中的输出接口表名（如 `PBOCD_JS_201_HDASZDKFS`）
- 引用的监管集市表列表

### 第三步：定位 INSERT INTO 语句

在文件中找到以下模式：

```sql
INSERT INTO PBOCD_DATACORE.PBOCD_JS_XXX
  (DATA_DATE,  -- 数据日期
   FIELD_TYPE,  -- 字段类别
   ...
  )
```

这就是输出到接口表的语句。目标字段列表在括号中，按列顺序对应SELECT。

### 第四步：查找对应的 SELECT 列表

INSERT INTO 之后紧跟着的 SELECT 语句。SELECT 表达式的列顺序对应 INSERT INTO 括号中的目标字段顺序。

### 第五步：查监管集市表定义

从 `参考资料/监管集市/` 下找到源码文件引用的表定义文件（如 `065_SMTMODS_L_ACCT_LOAN_贷款借据信息表.md`），确认来源字段的中文名。

### 第六步：添加注释

在 SELECT 列表的每个表达式末尾添加行内注释，格式为：

```
表达式, -- → 目标字段  来源说明（转换逻辑）
```

规则：
1. **简单映射**：直接使用的字段加来源表中文名
   ```sql
   T.CUST_ID, -- → CUST_ID  客户号（L_ACCT_LOAN.CUST_ID）
   ```
2. **固定值**：标注固定值和含义
   ```sql
   'T99', -- → FIELD_TYPE  字段类别，固定值T99=逐笔明细
   ```
3. **CASE WHEN**：在 END 后面加注释，说明分支逻辑
   ```sql
   END, -- → BIZ_TYPE  业务类别（非贴现=C01单位贷款，贴现=C03票据融资）
   ```
4. **计算表达式**：标注计算公式
   ```sql
   T.LOAN_ACCT_BAL * U.CCY_RATE, -- → LOAN_ACCT_BAL_RMB  贷款余额折人民币（余额×汇率）
   ```
5. **复杂聚合**：在 AS 别名后加注释
   ```sql
   SUM(...) AS BALANCE_SUM, -- → BALANCE_SUM  贷款汇总金额（余额×汇率，贴现则放款金额-贴现利息）
   ```

### 第七步：验证

修改后读取被修改的行，确认：
- 每个SELECT表达式都有 `-- → 目标字段` 注释
- 目标字段名与 INSERT INTO 括号中的字段名一致
- 来源说明引用了正确的监管集市表中文名

## 常见模式

### 模式A：单层INSERT-SELECT（T99逐笔明细）

INSERT 目标字段数量多（20-30个），SELECT 使用 JOIN 关联多表。注释重点放在 JOIN 表别名对应的实体。

### 模式B：嵌套子查询（T01/T02-T05汇总）

外层 SELECT 从子查询聚合结果中取值。注释需要同时标外层（直接映射）和内层（计算逻辑）。

### 模式C：临时表中间步骤

先 INSERT 到 TMP1 临时表，再从临时表 INSERT 到接口表。临时表字段的注释可简略，接口表输出的注释要详细。

## 注意事项

1. **准确性优先**：每个字段的映射关系必须从代码中验证，不能凭猜测
2. **区分同名不同源**：注意 JOIN 中同名字段的不同来源表（如 `A.ORG_NUM` vs `C.ORG_NUM`）
3. **临时表列名断层**：临时表 SELECT 列名可能与后续引用名不同（如 TMP1 中 `DIGITAL_ECONOMY_INDUSTRY` 被后续引用为 `FLAG`），需要在注释中说明
4. **INSERT 列顺序**：确认 INSERT INTO 括号中的列顺序，确保 SELECT 列顺序一一对应
5. **保留原有注释**：不删除或覆盖已有的中文注释，在原有注释基础上增加 `→ 目标字段` 映射说明
