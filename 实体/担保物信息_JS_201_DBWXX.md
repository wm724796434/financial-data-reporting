# 第一部分：发文原文要求（人行规范层）

> **用于回答"这个表要求报送什么"、"新增业务应该报到哪个表"等问题**

## 1. 官方报送范围

担保物是指借款人或第三人向金融机构提供的，用于担保债务履行的抵押物、质押物等财产。本表报送金融机构期末存续的担保物信息。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 按官方报送范围定义的相关担保物。 |
| ❌ 排除 | 核销客户对应的担保物（已核销贷款的担保物不计入）。 |

## 3. 关键业务要素

报送机构识别；担保合同编号；抵质押物类型及价值；评估信息；质押率等。

## 3. 业务筛选条件

### 3.1 核销客户剔除

```sql
-- 标记核销客户
INSERT INTO JS_102_FTYKHX_TEMP03
  SELECT T.CUST_ID, COUNT(1) BS
    FROM SMTMODS.L_ACCT_LOAN T
   WHERE T.CANCEL_FLG = 'Y'         -- 码表A0010（是否标志）：Y=是（已核销，标记为核销客户）；N=否（未核销）
   GROUP BY T.CUST_ID;
```

### 3.2 贷款余额汇总

```sql
WHERE T.LOAN_STOCKEN_DATE IS NULL   -- 资产未转让
GROUP BY T.ACCT_NUM
HAVING SUM(LOAN_ACCT_BAL) > 0      -- 余额大于0
```

### 3.3 担保物信息

从`SMTMODS.L_AGRE_GUARANTY_INFO`取抵质押物详细信息（类型、评估价值、抵押率等），通过`L_AGRE_GUARANTEE_RELATION`关联担保合同。

## 4. 特殊处理规则

### 4.1 担保物唯一性

通过`GUARANTEE_DBW_TMP01`中间表对同一合同下多笔押品按行号排序（RN），取最先登记的押品信息。
