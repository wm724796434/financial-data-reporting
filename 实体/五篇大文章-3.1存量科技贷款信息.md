# 第一部分：发文原文要求（人行规范层）

> **用于回答"这个表要求报送什么"、"新增业务应该报到哪个表"等问题**

## 1. 官方报送范围

科技贷款是指金融机构向符合科技型企业标准的借款人发放的各类贷款。本表报送报告期末存续的科技贷款存量数据，按科技贷款标识口径统计。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 符合科技型企业标识（通过科技名单PUB_KJDK识别）的存量贷款，含票据直贴 |
| ❌ 排除 | 无科技标识的常规贷款 |

## 3. 关键业务要素

报送维度：科技标识分类、所属行业阶段、企业规模、高新技术类型、贷款余额及利率等。

## 4. 业务筛选条件

### 4.1 科技企业识别

通过`SMTMODS.PUB_KJDK`（科技贷款名单）匹配存量贷款客户，识别科技型企业。

### 4.2 存量数据过滤

```sql
WHERE T.LOAN_ACCT_BAL > 0
  AND T.CANCEL_FLG <> 'Y'  -- 码表A0010（是否标志）：Y=是（已核销），排除；保留N=否（未核销）
  AND T.LOAN_STOCKEN_DATE IS NULL  -- 资产证券化日期为空，即资产未转让
```

## 5. 特殊处理规则

### 5.1 科技标识分类

通过科技企业名单（PUB_KJDK）进行匹配，按产业结构类型（indust_stg_type）、高新技术制造业（high_tech_mnft）、高新技术服务业（high_tech_srve）、专利密集型产业（pant_dens_indu）等多维度分类。

### 5.2 应用层汇总修正

详见应用层特殊处理 SQL：`五大篇章-18科技贷款刷汇总金额利率.sql`

该 SQL 对存量科技贷款数据进行：
- 票据编号校验（对应存量票据融资信息）
- 合同-借据关联校验（对应存量单位贷款信息）
- 机构号和机构代码刷写
