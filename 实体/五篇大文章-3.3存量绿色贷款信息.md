# 第一部分：发文原文要求（人行规范层）

> **用于回答"这个表要求报送什么"、"新增业务应该报到哪个表"等问题**

## 1. 官方报送范围

绿色贷款是指金融机构向符合绿色产业标准的项目或借款人发放的贷款。本表报送报告期末存续的绿色贷款存量数据，含直贷和票据直转两种模式。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 期末存续且绿色贷款标识不为空的贷款业务（含直贷和直转票据） |
| ❌ 排除 | 无绿色标识的贷款、已核销贷款、已转让贷款 |

## 3. 关键业务要素

报送维度：存量余额、客户信息、机构信息、绿色用途分类、加权平均利率等。

## 4. 取数筛选条件

### 绿色贷款识别与存量过滤（合并条件）

以下条件全部通过 AND 连接，**同时满足才算一条存量绿色贷款**：

```sql
WHERE T.GREEN_CREDIT_FLAG = '1'       -- 绿色信贷标志(2024版)：1=是
  AND T.LOAN_ACCT_BAL > 0             -- 余额大于0
  AND T.ACCT_STS <> '3'               -- 非结清状态。码表A0005：3=结清（排除），保留1=正常/2=逾期/9=其他
  AND T.CANCEL_FLG <> 'Y'             -- 未核销。码表A0010：Y=是（已核销），排除；保留N=否
  AND T.LOAN_STOCKEN_DATE IS NULL     -- 未证券化
```

**说明：**
- `GREEN_CREDIT_FLAG`（是否绿色信贷2024版）是唯一参与筛选的字段，取值 `'1'`
- `GREEN_CREDIT_USAGE`（绿色信贷用途分类2024版）和 `GREEN_CREDIT_RATIO`（绿色信贷贷款用途占比2024版）**不参与筛选**，仅在 SELECT 中透传到输出表，用于后续分类映射和数值输出
- `ACCT_STS` 的码表：1=正常 / 2=逾期 / 3=结清（排除） / 9=其它

### 直转票据识别

通过自连接`L_ACCT_LOAN`区分直贷与直转票据：
- 直转票据：`ITEM_CD` 在直贴(130101/130104)和转贴(130102/130105)科目范围内
- 直贷模式：非直转票据归入此类别

## 5. 特殊处理规则

存量数据分为直贷和直转两部分，通过 UNION ALL 合并输出。
