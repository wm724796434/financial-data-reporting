# 第一部分：发文原文要求（人行规范层）

## 1. 官方报送范围

绿色贷款是指金融机构向符合绿色产业标准的项目或借款人发放的贷款。本表报送绿色贷款的发生额信息，按绿色贷款指标口径逐笔统计报告期内的贷款发放金额（含票据直转）。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 符合绿色贷款标识的贷款业务，含直接发放和票据直转两种形式 |
| ❌ 排除 | 无绿色贷款标识的常规贷款 |

## 3. 关键业务要素

报送维度：客户维度、机构维度、产品维度、绿色标识维度、汇总金额及利率等。

## 4. 取数筛选条件

### 绿色贷款识别与发生额时间范围（合并条件）

以下条件全部通过 AND 连接，**同时满足才算一条绿色贷款发生额记录**：

```sql
WHERE T.GREEN_CREDIT_FLAG = '1'                       -- 绿色信贷标志(2024版)：1=是
  AND TO_CHAR(T.DRAWDOWN_DT, 'YYYYMM') = SUBSTR(IS_DATE, 1, 6)  -- 放款日期在报告期月份
```

**说明：**
- `GREEN_CREDIT_FLAG`（是否绿色信贷2024版）是唯一参与筛选的字段，取值 `'1'`
- 发生额按**放款日期**（DRAWDOWN_DT）过滤，不判断余额/核销/结清等存量条件（因为发生额看的是当期新发放，不是期末存续）
- `GREEN_CREDIT_USAGE` 和 `GREEN_CREDIT_RATIO` **不参与筛选**，仅在 SELECT 中用于后续处理

### 绿色用途分类映射（非筛选，仅映射）

`GREEN_CREDIT_USAGE` 取首位数字映射到输出表的 `FIELD_TYPE`：

```sql
CASE WHEN A2.INLANDORRSHORE_FLG = 'N' AND T.GREEN_CREDIT_USAGE IS NOT NULL THEN 'TW00'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '1' THEN 'TNA01'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '2' THEN 'TNA02'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '3' THEN 'TNA03'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '4' THEN 'TNA04'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '5' THEN 'TNA05'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '6' THEN 'TNA06'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '7' THEN 'TNA07'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '8' THEN 'TNB'
     WHEN SUBSTR(T.GREEN_CREDIT_USAGE, 1, 1) = '9' THEN 'TNC'
END AS FIELD_TYPE
```

- 映射来源：`GREEN_CREDIT_USAGE` 首位数字 → TNA01~TNA07 / TNB / TNC
- 离岸客户（`INLANDORRSHORE_FLG = 'N'`）且用途不为空 → TW00
- `GREEN_CREDIT_RATIO`（绿色信贷比例）直接输出到输出表的 `GREEN_LOAN_RATIO` 字段。历史版本曾做 `ROUND(GREEN_CREDIT_RATIO/100,5)`，2025年12月改为直取。

### 直转票据判断

将直转票据（非直贷）单独标识：
```sql
INNER JOIN SMTMODS.L_ACCT_LOAN T1
   ON T.ACCT_NUM || T.DRAFT_RNG = T1.ACCT_NUM || T1.DRAFT_RNG
  AND SUBSTR(T1.ITEM_CD, 1, 6) IN ('130101', '130104')  -- 直贴票据科目
  AND SUBSTR(T.ITEM_CD, 1, 6) IN ('130102', '130105')   -- 转贴现科目
```
数据标记为`直转`类型。

### 4.4 非直转票据

非直转票据（直贷模式）通过 UNION ALL 并入结果集，无需 JOIN。
