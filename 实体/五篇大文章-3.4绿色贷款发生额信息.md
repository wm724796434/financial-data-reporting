# 第一部分：发文原文要求（人行规范层）

> **用于回答"这个表要求报送什么"、"新增业务应该报到哪个表"等问题**

## 1. 官方报送范围

绿色贷款是指金融机构向符合绿色产业标准的项目或借款人发放的贷款。本表报送绿色贷款的发生额信息，按绿色贷款指标口径逐笔统计报告期内的贷款发放金额（含票据直转）。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 符合绿色贷款标识的贷款业务，含直接发放和票据直转两种形式 |
| ❌ 排除 | 无绿色贷款标识的常规贷款 |

## 3. 关键业务要素

报送维度：客户维度、机构维度、产品维度、绿色标识维度、汇总金额及利率等。

## 4. 业务筛选条件

### 4.1 绿色贷款标识

通过`L_ACCT_LOAN`表中的绿色贷款相关字段识别绿色贷款：
- `GREEN_CREDIT_FLAG` — 绿色信贷标识
- `GREEN_CREDIT_USAGE` — 绿色信贷用途
- `GREEN_CREDIT_RATIO` — 绿色信贷比例

### 4.2 发生额时间范围

```sql
WHERE TO_CHAR(T.DRAWDOWN_DT, 'YYYYMM') = SUBSTR(IS_DATE, 1, 6)
```
按报告期月份筛选贷款放款日期。

### 4.3 直转票据判断

将直转票据（非直贷）单独标识：
```sql
INNER JOIN SMTMODS.L_ACCT_LOAN T1
   ON T.ACCT_NUM || T.DRAFT_RNG = T1.ACCT_NUM || T1.DRAFT_RNG
  AND SUBSTR(T1.ITEM_CD, 1, 6) IN ('130101', '130104')  -- 直贴票据科目
  AND SUBSTR(T.ITEM_CD, 1, 6) IN ('130102', '130105')   -- 转贴现科目
```
数据标记为`直转`类型。

### 4.4 非直转票据

非直转票据（直贷模式）通过 UNION ALL 并入结果集，无需 JOIN。
