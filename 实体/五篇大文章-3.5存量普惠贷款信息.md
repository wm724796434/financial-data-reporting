# 第一部分：发文原文要求（人行规范层）

> **用于回答"这个表要求报送什么"、"新增业务应该报到哪个表"等问题"

## 1. 官方报送范围

普惠贷款是指金融机构向普惠群体（小微企业主、个体工商户、农户等）发放的贷款。本表报送报告期末存续的普惠贷款存量数据。

## 2. 纳入与排除口径

| 方向 | 内容 |
|------|------|
| ✅ 纳入 | 符合普惠标识的存量贷款，含个人经营性贷款 |
| ❌ 排除 | 非普惠群体贷款、大额企业贷款 |

## 3. 关键业务要素

报送维度：客户类型（小微企业/个体工商/农户）、贷款余额、利率、担保方式等。

## 4. 业务筛选条件

### 4.1 普惠客户识别

通过对公客户类型（L_CUST_C）和个人客户经营类型（L_CUST_P）识别普惠群体：
- 小微企业：对公客户中小微企业标识
- 个体工商户：个人客户中经营类型为'AC'（个体工商户）
- 小微企业主：个人客户中经营类型为'B'
- 随借随还贷款：通过`V_PUB_IDX_DK_GRSNDK`视图识别

### 4.2 存量数据过滤

```sql
WHERE T.LOAN_ACCT_BAL > 0  -- 贷款余额大于0
  AND T.ACCT_STS <> '3'  -- 码表A0005（贷款账户状态）：3=结清（排除），保留1=正常/2=逾期/9=其他
  AND T.CANCEL_FLG <> 'Y'  -- 码表A0010（是否标志）：Y=是（已核销），排除；保留N=否（未核销）
  AND T.LOAN_STOCKEN_DATE IS NULL  -- 资产证券化日期为空，即资产未转让
```

### 4.3 贸易融资标识

通过`L_ACCT_TRAD_FIN`识别贸易融资类贷款。

## 5. 特殊处理规则

存量数据分为直贷和直转两部分，通过 UNION ALL 合并输出。应用层通过`五大篇章-普惠贷款刷汇总金额利率.sql`计算汇总金额和加权平均利率。
